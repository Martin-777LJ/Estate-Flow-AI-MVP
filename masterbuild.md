

You are building a production-ready SaaS-lite web application called **EstateFlow AI**.

## Project Summary
EstateFlow AI is an **AI chatbot + lead capture system for real estate businesses**.

It helps real estate businesses:
- answer customer questions instantly
- capture and qualify leads
- reduce missed website inquiries
- manage conversations and FAQs from a dashboard

## Build Goal
Build a clean, polished, modern MVP that looks and feels like a real product that could be shown to a paying client immediately.

This is **not** a hackathon toy or placeholder demo.

---

## Required Stack
- Next.js (App Router)
- TypeScript
- Tailwind CSS
- shadcn/ui
- Supabase (database + auth)
- Gemini API integration

---

## Product Structure
Build these main parts:

### Public Site
- landing page

### Admin App
- login
- dashboard overview
- leads page
- conversations page
- knowledge base page
- settings page
- install page

### Embedded Widget
- floating website chatbot
- lead capture flow
- FAQ answer handling

---

## UI / Design Direction
Use a premium B2B SaaS design style:
- clean layout
- rounded cards
- soft shadows
- excellent spacing
- responsive on mobile and desktop
- polished enough to sell to a client

Do not make it look childish, generic, or overly flashy.

---

## Product Constraints
Do NOT overbuild.
Do NOT add unnecessary enterprise complexity.

Do NOT add:
- billing
- multi-tenant complexity
- advanced CRM integrations
- complex analytics engine
- unnecessary backend architecture

Focus on:
- clean architecture
- demo quality
- useful UX
- client-ready polish
